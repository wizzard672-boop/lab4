## Условия задач

### Задача 1. Замыкание для получения простых чисел

**Требуется:**
Создать функцию `prime_generator()`, которая возвращает внутреннюю функцию-генератор. При каждом вызове внутренняя функция должна возвращать следующее простое число, начиная с 2.

**Особенности реализации:**

- Внутренняя функция должна хранить состояние (последнее найденное число и список уже вычисленных простых чисел)
- Проверка на простоту должна быть оптимизирована за счёт использования накопленного списка простых чисел

### 1. Реализация замыкания (`prime_generator`)

Создана внешняя функция prime_generator, которая содержит список primes и переменную current. Внутренняя функция get_next_prime использует nonlocal current для изменения состояния внешней функции. Алгоритм проверки на простоту использует уже найденные простые числа для сокращения количества делений.




### Задача 2. Декоратор ограничения времени выполнения

**Требуется:**
Создать параметризованный декоратор `@timeout(n)`, который ограничивает время выполнения декорируемой функции `n` секундами. При превышении лимита должно выбрасываться пользовательское исключение `TimeoutError`.

**Особенности реализации:**

- Использование модуля `signal` и сигнала `SIGALRM` (для Unix-систем)
- Корректное восстановление обработчиков сигналов после выполнения функции
- Обработка исключительных ситуаций в блоке `finally`

### 2. Реализация декоратора (`timeout`)

Декоратор реализован с использованием модуля `signal` (сигнал `SIGALRM`).

- **Важно:** Данная реализация работает в Unix-подобных системах (Linux, macOS) и может не работать в Windows без дополнительных эмуляторов или использования `threading`.
- Принцип работы: перед запуском функции устанавливается таймер. Если функция не завершилась за отведённое время, вызывается обработчик сигнала, который выбрасывает исключение `TimeoutError`.
- После завершения (или прерывания) таймер сбрасывается, и восстанавливается предыдущий обработчик сигнала.
